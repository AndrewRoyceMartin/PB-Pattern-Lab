Composite No-Frequency (benchmark-chosen, stable)

Optimised (auto-optimise → then generate)

And you should never be unsure whether the optimiser affected a pick again.

Button: Generate 12 Picks (Composite No-Frequency)

Uses the current validated best strategy (Composite without long-term frequency signals).

No optimiser involved.

Lane B — Optimised (Experimental)

Runs optimiser first, then generates.

Optimised lane: always runs optimiser first.

5) Explain “Composite No-Frequency” in plain English (simple)

Use this exact definition everywhere:

Composite No-Frequency = a composite scoring strategy that uses recency + trend + structure + set construction, but does not use long-term ‘most drawn’ frequency (because long-term frequency has not helped in validation).

That’s it. No extra detail required.

6) Make “Optimised” deterministic and safe
New backend endpoint (recommended)

Add a single endpoint so the frontend can’t mis-order calls:

POST /api/auto/optimise-and-generate

It does:

Load latest benchmark context (optional)

Run optimiser (with fixed config)

Generate 12 picks using the optimiser weights

Return picks + optimiser metadata + run stamp

✅ This ensures you never generate picks using “old weights”.

Also keep existing:

POST /api/auto/generate-composite-no-frequency (simple lane A)

Show only the two lanes + results.

“Advanced” accordion (collapsed by default)
