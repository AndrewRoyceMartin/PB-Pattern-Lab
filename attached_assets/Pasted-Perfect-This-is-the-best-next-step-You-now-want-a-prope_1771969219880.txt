What “better validation” means
1) Test multiple windows, not one

Run walk-forward backtests for:

20 draws

40 draws

60 draws

100 draws (if enough data)

This stops one lucky window from misleading you.

2) Compare against random every time

For each strategy and window, compute:

Delta vs Random for:

mean best main matches

mean best total score

Powerball hit rate

avg matches across top 10 cards

This gives you a clean benchmark.

3) Add stability scoring

A strategy is only interesting if it is consistent.

Example stability rules

Stable positive: beats random in 3+ windows

Weak/unstable: beats random in only 1 window

No edge: mostly equal to or below random

Underperforming: below random in most windows

4) Add significance checks (later optional, but recommended)

Once multi-window is running, add:

Permutation test (strategy score vs shuffled outcomes)

Monte Carlo benchmark (real results vs simulated fair histories)

This tells you if observed lift is beyond chance.

Step 6 — Add stability classification

After all windows finish, aggregate per strategy.

Classification logic (simple and effective)

For each strategy:

count windows where delta_best_total_score_vs_random > +0.05

count windows where delta_best_total_score_vs_random < -0.05

Then classify:

possible_edge = positive in >= 3 windows and never strongly negative

weak_edge = positive in 2 windows

no_edge = mixed / near zero

underperforming = negative in most windows

Use this as a top-line verdict.
