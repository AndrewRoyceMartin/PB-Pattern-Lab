Task 2 — Implement most_drawn_last_20 strategy
Scope

Add a short-window frequency strategy using only the last 20 training draws.

Logic

Training draws only (no leakage)

Use trainDraws.slice(-20)

Count main number frequencies (1–35)

Count Powerball frequencies (PB range in your rules)

Rank by descending count

Deterministic tie-breakers (count desc, recency desc, number asc)

Generate top 10 cards using existing candidate/generator pipeline

Acceptance criteria

Strategy returns valid top 10 cards

Cards use only training data counts

Appears in Generator page selector

Appears in Validation benchmark results

Diagnostics include diversity metrics

Task 3 — Implement least_drawn_last_50 strategy (contrarian baseline)
Scope

Add a contrarian benchmark using least frequent numbers from the last 50 training draws.

Logic

Use trainDraws.slice(-50)

Rank numbers by ascending frequency (least drawn first)

Tie-break deterministically

Generate top 10 cards

Keep diversity controls active (don’t collapse into near-identical cards)

Why this matters

This tests whether “cold numbers” perform any better/worse than “hot numbers” and strengthens your benchmark suite.

Acceptance criteria

Strategy returns valid cards

Included in validation benchmark and generator mode list

Delta vs random is shown in validation results

No leakage in walk-forward backtests

Task 4 — Implement structure_matched_random strategy
Scope

Add a random strategy constrained by historically common draw structure.

Logic (v1)

Use training draws to estimate typical structure ranges/distributions:

odd/even count

low/high split

sum range (e.g., central quantile band)

spread/range

consecutive count cap (optional)

decade distribution heuristic (optional v2)

Generate random candidate cards, then score/filter by structural fit:

Keep cards that match common structure

Rank by structure-fit score

Return top 10

Notes

This is a better baseline than plain random because it controls for draw “shape.”

Acceptance criteria

Valid top 10 cards returned

Validation shows it as a separate benchmark row

Summary table includes delta vs plain random

Strategy is reproducible (same seed/config => same output if you support seeding)

Task 5 — Implement anti_popular_only strategy (formal benchmark strategy)
Scope

Promote anti-popularity scoring into a standalone strategy that can be validated against random.

Logic

Score cards only by anti-popularity heuristics (no draw-fit signal):

penalty for many numbers in 1..31 (birthday bias)

sequence penalty (e.g., 1-2-3, long runs)

repeated ending-digit penalty

symmetry / neat-pattern penalty (simple heuristics)

optional “human-like card” score inversion

Output:

top 10 lowest predicted popularity cards (or highest anti-pop score)

Why this matters

Even if predictive edge is absent, this may be your strongest practical mode.

Acceptance criteria

Strategy appears in Generator + Validation

Validation benchmark can compare it vs random (even if predictive score is neutral)

Generator shows anti-popularity breakdown/explanation for cards

Recommendation engine can choose this mode when no predictive edge exists

Task 6 — Implement diversity_optimized strategy
Scope

Add a strategy that optimizes coverage/diversity across the top 10 generated cards.

Logic (v1)

Start from a base candidate pool (random, structure-matched, or composite-lite), then apply diversity penalties during selection:

penalize repeated Powerball values across selected top 10

penalize excessive overlap in main numbers with already selected cards

encourage wider number coverage across the set

maintain card validity + score quality threshold

Think of it as:

generate many candidates

greedily select 10 cards maximizing (score - overlap penalty)

Why this matters

Very useful for users buying multiple tickets.

Acceptance criteria

Returns 10 valid cards

Diversity metrics improve vs non-diversity strategies:

more unique PBs

lower average overlap

more unique mains across top 10

Included in validation benchmark + generator selector
